Here’s the information formatted as a clean **2-level nested bullet list**:

- **Pint**
  - 🔗 Base Link: https://github.com/laravel/pint
  - 📖 Resource: https://laravel.com/docs/pint
  - 💡 Why we use it: Official Laravel code formatter that automatically enforces PSR-12/Laravel coding standards with zero configuration, keeping code consistently styled across teams and ideal for CI/CD or pre-commit hooks.

- **Larastan**
  - 🔗 Base Link: https://github.com/larastan/larastan
  - 📖 Resource: https://larastan.dev/
  - 💡 Why we use it: Laravel-tuned static analysis tool (built on PHPStan) that catches type mismatches, broken facades, invalid Eloquent relationships, and missing service bindings before runtime, drastically reducing production bugs.

- **Rector**
  - 🔗 Base Link: https://github.com/rectorphp/rector
  - 📖 Resource: https://getrector.com/documentation
  - 💡 Why we use it: Automated refactoring and PHP/Laravel version upgrader that instantly modernizes legacy code, removes deprecated patterns, and applies rule-based code transformations with minimal manual effort.

- **Code Sniffer (PHPCS)**
  - 🔗 Base Link: https://github.com/PHPCSStandards/PHP_CodeSniffer
  - 📖 Resource: https://github.com/PHPCSStandards/PHP_CodeSniffer/wiki
  - 💡 Why we use it: Static analyzer that detects and reports coding standard violations (PSR-12, Laravel, custom rulesets) without auto-fixing, making it ideal for enforcing team-wide style rules in CI pipelines.

- **Laravel IDE Helper**
  - 🔗 Base Link: https://github.com/barryvdh/laravel-ide-helper
  - 📖 Resource: https://github.com/barryvdh/laravel-ide-helper#readme
  - 💡 Why we use it: Generates PHPDoc stubs for facades, models, macros, configs, and routes, bridging Laravel’s dynamic runtime features with static IDE support for accurate autocomplete, type hints, and "go-to-definition".

- **Laravel Extension (VS Code)**
  - 🔗 Base Link: https://marketplace.visualstudio.com/items?itemName=oneil.laravel
  - 📖 Resource: VS Code Marketplace & community documentation
  - 💡 Why we use it: Adds Laravel-specific IDE features like Blade syntax highlighting, Artisan command execution, route/model navigation, and `.env` validation, significantly speeding up daily development in VS Code.
